# German Weekly

A German-learning site published on GitHub Pages. Lessons are generated by
Python scripts (run by a GitHub Action), committed as JSON under
`public/lessons/`, and rendered by a Next.js/React frontend where every German
word is clickable for a dictionary lookup.

Two kinds of lesson:

- **Text** — an AI-written (or article-derived) lesson at a target CEFR level,
  with vocab, phrases, a quiz, and TTS audio.
- **Video** — a YouTube video with its German subtitles shown as a transcript
  that scrolls in sync with playback. Subtitles come from the video's own German
  captions, or a local transcription when the video has none.

## Layout

```
.                      Next.js app (App Router, static export)
  app/                 routes: / (lesson view), /deck
  components/          React components (lesson, video, quiz, lookup, deck…)
  hooks/  lib/         data fetching, types, deck storage, tokenizer, YT sync
  public/lessons/      generated lessons (JSON + MP3), served at /lessons/
  scripts/             Python lesson generators (unchanged logic)
```

## How it works

`scripts/germanweekly.py` is the entry point for generation. It parses arguments
and dispatches to one of two pipelines, importing only what that pipeline needs:

| Input | Pipeline | Lesson type |
|---|---|---|
| `--topic "Berlin Wall"` | `scripts/text_lesson.py` — LLM writes a fresh lesson | `text` |
| `--source <article URL>` | `scripts/text_lesson.py` — LLM rewrites the page at the level | `text` |
| `--youtube <video URL>` | `scripts/video_lesson.py` — captions, else local transcription | `video` |

- Each lesson is one file: `public/lessons/<id>.json` (text lessons also get
  `<id>.mp3`). The id is a date + title slug, e.g.
  `2026-05-27-was-machst-du-mit-deinem-leben`.
- `public/lessons/index.json` lists every lesson newest-first with just the
  metadata the frontend needs (`id`, `type`, `title`, `date`, `level`,
  `video_id`). `scripts/lessons_io.py` (`save_lesson`) writes the lesson file and
  upserts this index.
- The Next.js app fetches `/lessons/index.json` at runtime, then renders one
  lesson by `type`: `?lesson=<id>` selects a specific lesson; with no parameter
  it shows the newest. A video lesson embeds the YouTube player beside a synced,
  clickable transcript; a text lesson renders the article, vocab, phrases, and
  quiz.
- A left **drawer** (hamburger in the nav) lists all lessons so you can browse
  and open previous ones.
- Clicking any German word opens a dictionary lookup (hosted at
  `https://dict.germanweekly.com/api/lookup/`) with a **Save to deck** button.
- The **`/deck`** route is a personal vocabulary deck stored in the browser
  (`localStorage`, single user, no accounts).
- `.github/workflows/generate-lesson.yml` runs the generator weekly (Monday) or
  on demand and commits the new lesson file plus the updated `index.json`.
- `.github/workflows/deploy.yml` builds the static export and publishes it to
  GitHub Pages on every push to `main`, so a freshly committed lesson goes live
  automatically.

### Video subtitles: captions first, local transcription as fallback

`scripts/video_lesson.py` tries the video's German captions via `yt-dlp`
(manually uploaded subtitles first, then YouTube's auto-captions), parsing them
into timed segments. Auto-captions arrive in YouTube's "rolling" format (each cue
repeats the previous line); the parser strips inline tags and de-duplicates them.

If the video has no German captions, it falls back to a local transcription with
[`yapsnap`](https://github.com/kouhxp/yapsnap) using the **German Kroko model**.
The model directory is passed via the `KROKO_MODEL` environment variable (the
default `yapsnap` model is English and will not work for German).

## Lesson JSON

Each lesson is stored as `public/lessons/<id>.json`.

**Text lesson**:

```json
{
  "type": "text",
  "id": "2026-05-27-die-berliner-mauer",
  "title": "...",
  "text": "...",
  "date": "2026-05-27T...",
  "level": "B1",
  "topic": "...",
  "source": { "type": "ai" },
  "vocabs":  [ { "word": "...", "definition": "...", "example": "..." } ],
  "phrases": [ { "phrase": "...", "translation": "...", "example": "..." } ],
  "quiz":    [ { "type": "mcq" | "cloze", ... } ],
  "voice_path": "lessons/2026-05-27-die-berliner-mauer.mp3"
}
```

**Video lesson**:

```json
{
  "type": "video",
  "id": "2026-05-27-was-machst-du-mit-deinem-leben",
  "title": "...",
  "video_id": "w9J6D4r30HY",
  "url": "https://www.youtube.com/watch?v=...",
  "date": "2026-05-27T...",
  "level": "B1",
  "source": { "type": "youtube", "url": "..." },
  "transcript_source": "captions" | "yapsnap",
  "segments": [ { "start": 11.0, "end": 15.0, "text": "..." } ]
}
```

`voice_path` is a web path served at `/lessons/<id>.mp3`. `end` is present for
caption-sourced segments; for `yapsnap` segments only `start` is set and the
player uses the next segment's start as the boundary.

## Local development

### Frontend

```bash
pnpm install
pnpm dev            # http://localhost:3000
pnpm build          # static export to ./out
```

### Generating lessons

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r scripts/requirements.txt
cp .env.example .env          # add your API keys

python scripts/germanweekly.py --topic "Die Berliner Mauer"        # AI-written, B1 by default
python scripts/germanweekly.py --source https://de.wikipedia.org/wiki/...   # rewrite an article
python scripts/germanweekly.py --level B2 --topic "..." --no-audio          # other options
```

Video lessons (need `ffmpeg` on PATH plus the German model):

```bash
export KROKO_MODEL=/path/to/sherpa-onnx-streaming-zipformer-de-kroko-...   # German model dir

python scripts/germanweekly.py --youtube "https://www.youtube.com/watch?v=w9J6D4r30HY"
```

Generators write into `public/lessons/`, so a new lesson shows up at
`http://localhost:3000` immediately while `pnpm dev` is running.

Run the parser tests:

```bash
cd scripts && python -m unittest test_video_lesson
```

## GitHub Action

The generate workflow has two jobs in `.github/workflows/generate-lesson.yml`:

- **text** — runs on the weekly schedule (Mondays 08:00 UTC) and for manual runs
  without a YouTube URL. Uses `scripts/topics-queue.txt` when no topic/source is
  given.
- **video** — runs only for a manual run that sets the `youtube_url` input. It
  installs `ffmpeg`, downloads and caches the German Kroko model, then builds the
  video lesson.

Run manually: **Actions → Generate Lesson → Run workflow**. Set `youtube_url` for
a video lesson, or `topic`/`source` for a text lesson. Committing the lesson to
`main` triggers the deploy workflow, which rebuilds and publishes the site.

Repository secrets (text lessons only):

- `OPENAI_API_KEY` (required)
- `ELEVENLABS_API_KEY` (required for ElevenLabs TTS)
- `OPENAI_MODEL`, `OPENAI_TTS_MODEL`, `OPENAI_TTS_VOICE` (optional)
- `ELEVENLABS_VOICE_ID`, `ELEVENLABS_MODEL_ID`, `ELEVENLABS_OUTPUT_FORMAT` (optional)

## GitHub Pages

1. In repository settings, set **Pages → Build and deployment → Source** to
   **GitHub Actions**.
2. Push to `main`. The `deploy.yml` workflow builds the static export and
   publishes it; the `CNAME` file keeps the `germanweekly.com` domain.

## Notes

- The vocabulary deck lives in the browser only — it is per-device and not synced.
- TTS tries ElevenLabs first and falls back to OpenAI TTS; the ElevenLabs voice is
  random unless `--voice-id` is passed.
- Video lessons make no LLM calls — the vocabulary is whatever you click and save.

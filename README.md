# German Weekly

Weekly German-learning newsletter generated by GitHub Actions and published on GitHub Pages.

## How it works

- `germanweekly.py` generates one CEFR-level lesson text (`--level`), vocab list, and audio.
- Active lesson is always saved to `lessons/latest.json` and `lessons/latest.mp3`.
- Previous lessons are moved automatically to `lessons/archive/`.
- `index.html` reads `lessons/latest.json` and renders the latest lesson page.
- `.github/workflows/generate-lesson.yml` runs weekly (Monday) or manually and commits updates.

## Lesson JSON format

`lessons/latest.json`:

```json
{
  "title": "...",
  "text": "...",
  "date": "2026-03-20T...",
  "category": "history",
  "vocabs": [
    { "word": "...", "definition": "..." }
  ],
  "voice_path": "lessons/latest.mp3"
}
```

## Local development

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python germanweekly.py --level B1
```

Optional flags:

- `--voice-id <voice_id>`
- `--dry-run`

## GitHub Action setup

Add these repository secrets in GitHub (`Settings` → `Secrets and variables` → `Actions`):

- `OPENAI_API_KEY` (required)
- `ELEVENLABS_API_KEY` (required for ElevenLabs TTS)
- `OPENAI_MODEL` (optional)
- `OPENAI_TTS_MODEL` (optional)
- `OPENAI_TTS_VOICE` (optional)
- `ELEVENLABS_VOICE_ID` (optional fallback)
- `ELEVENLABS_MODEL_ID` (optional)
- `ELEVENLABS_OUTPUT_FORMAT` (optional)

Then run:

- Manual: Actions → `Generate Weekly Lesson` → `Run workflow`
- Scheduled: every Monday at 08:00 UTC (already configured)

## GitHub Pages

1. Push repository to GitHub.
2. In repository settings, enable Pages from the default branch root.
3. Open the Pages URL; `index.html` shows the latest lesson automatically.

## Notes

- Category is chosen randomly each run.
- TTS tries ElevenLabs first; if it fails, it automatically falls back to OpenAI TTS.
- ElevenLabs voice is chosen randomly unless `--voice-id` is passed.
- OpenAI and ElevenLabs use their official Python SDKs.

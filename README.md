# German Weekly

A German-learning site published on GitHub Pages. Lessons are generated by a
GitHub Action, committed as JSON, and rendered by a static frontend where every
German word is clickable for a dictionary lookup.

Two kinds of lesson:

- **Text** â€” an AI-written (or article-derived) lesson at a target CEFR level,
  with vocab, phrases, a quiz, and TTS audio.
- **Video** â€” a YouTube video with its German subtitles shown as a transcript
  that scrolls in sync with playback. Subtitles come from the video's own German
  captions, or a local transcription when the video has none.

## How it works

`germanweekly.py` is the entry point. It parses arguments and dispatches to one
of two pipelines, importing only what that pipeline needs:

| Input | Pipeline | Lesson type |
|---|---|---|
| `--topic "Berlin Wall"` | `text_lesson.py` â€” LLM writes a fresh lesson | `text` |
| `--source <article URL>` | `text_lesson.py` â€” LLM rewrites the page at the level | `text` |
| `--youtube <video URL>` | `video_lesson.py` â€” captions, else local transcription | `video` |

- Each lesson is one file: `lessons/<id>.json` (text lessons also get
  `lessons/<id>.mp3`). The id is a date + title slug, e.g.
  `2026-05-27-was-machst-du-mit-deinem-leben`.
- `lessons/index.json` lists every lesson newest-first with just the metadata the
  frontend needs (`id`, `type`, `title`, `date`, `level`, `video_id`).
  `lessons_io.py` (`save_lesson`) writes the lesson file and upserts this index.
- `index.html` reads `index.json`, then renders one lesson by `type`:
  `?lesson=<id>` selects a specific lesson; with no parameter it shows the newest.
  A video lesson embeds the YouTube player beside a synced, clickable transcript;
  a text lesson renders the article, vocab, phrases, and quiz.
- A left **drawer** (hamburger in the nav) lists all lessons from `index.json` so
  you can browse and open previous ones.
- Clicking any German word opens a dictionary lookup (hosted at
  `https://dict.germanweekly.com/api/lookup/`) with a **Save to deck** button.
- `deck.html` is a personal vocabulary deck stored in the browser
  (`localStorage`, single user, no accounts). `deck.js` is the shared storage
  layer.
- `.github/workflows/generate-lesson.yml` runs the generator weekly (Monday) or
  on demand and commits the new lesson file plus the updated `index.json`.

### Video subtitles: captions first, local transcription as fallback

`video_lesson.py` tries the video's German captions via `yt-dlp` (manually
uploaded subtitles first, then YouTube's auto-captions), parsing them into timed
segments. Auto-captions arrive in YouTube's "rolling" format (each cue repeats
the previous line); the parser strips inline tags and de-duplicates them.

If the video has no German captions, it falls back to a local transcription with
[`yapsnap`](https://github.com/kouhxp/yapsnap) using the **German Kroko model**.
The model directory is passed via the `KROKO_MODEL` environment variable (the
default `yapsnap` model is English and will not work for German).

## Lesson JSON

Each lesson is stored as `lessons/<id>.json`.

**Text lesson**:

```json
{
  "type": "text",
  "id": "2026-05-27-die-berliner-mauer",
  "title": "...",
  "text": "...",
  "date": "2026-05-27T...",
  "level": "B1",
  "topic": "...",
  "source": { "type": "ai" },
  "vocabs":  [ { "word": "...", "definition": "...", "example": "..." } ],
  "phrases": [ { "phrase": "...", "translation": "...", "example": "..." } ],
  "quiz":    [ { "type": "mcq" | "cloze", ... } ],
  "voice_path": "lessons/2026-05-27-die-berliner-mauer.mp3"
}
```

**Video lesson**:

```json
{
  "type": "video",
  "id": "2026-05-27-was-machst-du-mit-deinem-leben",
  "title": "...",
  "video_id": "w9J6D4r30HY",
  "url": "https://www.youtube.com/watch?v=...",
  "date": "2026-05-27T...",
  "level": "B1",
  "source": { "type": "youtube", "url": "..." },
  "transcript_source": "captions" | "yapsnap",
  "segments": [ { "start": 11.0, "end": 15.0, "text": "..." } ]
}
```

`end` is present for caption-sourced segments; for `yapsnap` segments only
`start` is set and the player uses the next segment's start as the boundary.

## Local development

Text lessons:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # add your API keys

python germanweekly.py --topic "Die Berliner Mauer"        # AI-written, B1 by default
python germanweekly.py --source https://de.wikipedia.org/wiki/...   # rewrite an article
python germanweekly.py --level B2 --topic "..." --no-audio          # other options
```

Video lessons (need `ffmpeg` on PATH plus the video dependencies):

```bash
pip install -r requirements.txt
export KROKO_MODEL=/path/to/sherpa-onnx-streaming-zipformer-de-kroko-...   # German model dir

python germanweekly.py --youtube "https://www.youtube.com/watch?v=w9J6D4r30HY"
```

Serve the site to view a lesson:

```bash
python -m http.server 8000        # then open http://localhost:8000
```

Run the parser tests:

```bash
python -m unittest test_video_lesson
```

## GitHub Action

The workflow has two jobs in `.github/workflows/generate-lesson.yml`:

- **text** â€” runs on the weekly schedule (Mondays 08:00 UTC) and for manual runs
  without a YouTube URL. Uses `topics-queue.txt` when no topic/source is given.
- **video** â€” runs only for a manual run that sets the `youtube_url` input. It
  installs `ffmpeg` and `requirements-video.txt`, downloads and caches the German
  Kroko model, then builds the video lesson.

Run manually: **Actions â†’ Generate Lesson â†’ Run workflow**. Set `youtube_url` for
a video lesson, or `topic`/`source` for a text lesson.

Repository secrets (text lessons only):

- `OPENAI_API_KEY` (required)
- `ELEVENLABS_API_KEY` (required for ElevenLabs TTS)
- `OPENAI_MODEL`, `OPENAI_TTS_MODEL`, `OPENAI_TTS_VOICE` (optional)
- `ELEVENLABS_VOICE_ID`, `ELEVENLABS_MODEL_ID`, `ELEVENLABS_OUTPUT_FORMAT` (optional)

## GitHub Pages

1. Push the repository to GitHub.
2. In repository settings, enable Pages from the default branch root.
3. Open the Pages URL; `index.html` shows the latest lesson automatically.

## Notes

- The vocabulary deck lives in the browser only â€” it is per-device and not synced.
- TTS tries ElevenLabs first and falls back to OpenAI TTS; the ElevenLabs voice is
  random unless `--voice-id` is passed.
- Video lessons make no LLM calls â€” the vocabulary is whatever you click and save.
